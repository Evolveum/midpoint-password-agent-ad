# MidPoint Password Agent for Active Directory Listener

## Overview

The MidPoint Password Agent for Active Directory Listener is a Windows LSA Password Notification Filter implemented as a native C++ DLL (`MidPointPasswordAgentListener.dll`). It is loaded directly into the `lsass.exe` process on every Active Directory Domain Controller where it is deployed.

Its **sole responsibility** is to capture a cleartext password at the moment of change and persist it to a local filesystem queue as quickly as possible.

---

## LSA Filter Contract

The DLL exports three functions required by the LSA notification filter interface. All three must appear in `PasswordFilter.def` and be exported by name.

| Export | Purpose |
|---|---|
| `InitializeChangeNotify` | Called once when the DLL is loaded by LSA. Returns `TRUE`. |
| `PasswordFilter` | Called to evaluate whether a proposed password meets complexity requirements. Always returns `TRUE` — this filter imposes no additional requirements. |
| `PasswordChangeNotify` | Called after a password change has been committed to the directory. This is where the event is captured. |

> **Critical:** `PasswordChangeNotify` must always return `STATUS_SUCCESS`. Returning an error does not undo the password change — it only causes LSA to log a warning. However, an **unhandled exception** escaping this function will crash `lsass.exe` and trigger an immediate DC reboot.

## Build

### Prerequisites

- Visual Studio 2022 (or Build Tools) with the **Desktop development with C++** workload.
- CMake ≥ 4.3.0
- Windows SDK ≥ 10.0.20348 (Windows Server 2022 SDK).
- `spdlog` C++ logging library
- `vcpkg` C++ package manager (windows has included in Build Tools, mac has installation guide [here](../installation/mac.md#Install vcpkg))
- `nlohmann-json` C++ json parsing library

### Development

Use `clangd` for intellisese. Use `compile_commands.json` to help analyze dependencies as `spdlog` with

```bash
just listener-ide
```

### Logging

Listener application logs are divided into 2 categories:

- Application state logs - Windows EventLog (can be traces with Event Viewer Windows app)
- Application operations - stored in `.log` file path is defined [in](../../listener/src/logger/file_logger.cpp). Logged with cpp library `spdlog`

Location of `.log` files is configurable with Windows registry value located in `"HKLM:SOFTWARE\\Evolveum\\MidPointPasswordAgent\\LogPath"`.  Log files are created on daily bases. So there is separate log file for each day.

### File Queue

Used as IPC(Inter process communication) between Listener and Sender modules. When listener receives password change it stores this event into file in configurable location from Windows registry located in `"HKLM:SOFTWARE\\Evolveum\\MidPointPasswordAgent\\QueuePath"`. Queue files have `.event` extension and content is JSON.

```json
{ "username": <username>, "domain": <domain>, "password": <password> }
```

### File Encryption

For queue event file `password` field encryption uses `Listener` AES key generated by `Sender` module. Key sits in Windows Registry in `"HKLM:SOFTWARE\\Evolveum\\MidPointPasswordAgent\\Keys"`. Where one registry value points to latest key version.
```
"LatestKey": "v1",
"v1": <key>
```
Key is also protected with DPAPI. So before encryption of event file we need to uprotect it. This means that `.event` file is human readable JSON with only encrypted `password` field.

## Configuration

Listener module is configurable with `config.json` file. Its location is read from Windows Registry `"HKLM:SOFTWARE\Evolveum\MidPointPasswordAgent\ConfigPath"`. If configuration file does not exists or cannot be read defaults are used that are defined inside [`SenderJsonConfiguration`](../../sender/Configuration/SenderJsonConfiguration.cs)

Example of `config.json`
```json
{
    "Listener": {
        "LogLevel": "info",
        "LogFileSizeLimitBytes": 532480,
        "LogRetainedFileCountLimit": 3,
    }
}
```

`LogLevel` - Logging level [trace, debug, info, warn, err, critical, off]

`LogFileSizeLimitBytes` - Max size of log file in bytes (5MB default)

`LogRetainedFileCountLimit` - Max number or last log files to keep (3 default)
